## 4. 逻辑一致性检查

- 七个页面均在 `MainWindow::createPages()` 创建，并注册到 `NavigationService`。
- 侧边导航包含 Dashboard、Realtime Tags、Trend、Alarm Center、History、Measurement Map、Logs & Settings 七项。
- `MainWindow::refreshShellClock()` 每次通过 `UiSnapshotProvider::refresh()` 获取统一快照，再调用所有页面 `refresh(snapshot)`，符合“UI 通过快照刷新”的迁移原则。
- Start/Stop 会更新 `UiSnapshotProvider` 运行状态，停止后快照保留最后帧；已有 `UiSnapshotStartStop` 测试覆盖。
- CSV 导出使用统一 `CsvExportWriter`，Trend、History、Measurement Map 页面均有导出入口。
- 目前 UI 数据源是演示快照源，不直接查询真实 SQLite 仓储；这避免了 UI 直接读库，但也意味着 History/Logs/Settings 的完整持久化交互尚未闭合。

## 5. 后续建议

1. Dashboard 增加趋势预览控件，复用 `TrendChartWidget` 或轻量 Sparkline。
2. History 页面接入 `SQLiteHistoryRepository::query()`，补 page/pageSize、上一页/下一页、取消查询。
3. Alarm Center 历史区接入 `SQLiteAlarmRepository::query()`，补时间范围、状态/级别过滤和分页。
4. Logs & Settings 接入 `SQLiteOperationLogRepository` 和 `SQLiteConfigurationRepository`，实现日志查询条件、配置保存后可重载。
5. 增加 Qt UI smoke/交互测试，覆盖七页导航、Start/Stop、确认警报、CSV 导出和热力图点击选择。
