# 阶段 6：事件、队列和运行时线程迁移

执行日期：2026-06-25

## 完成范围

- 新增 `application/events/ApplicationEvents.*`，定义 RawFrame、TagRuntimeStates、Alarm、DataSourceHealth 等 Application 事件值对象。
- 扩展 `application/events/EventBus.*`，支持真实 handler 注册和同步发布，保留 Critical / Isolated 失败策略。
- 新增 `application/queues/BlockingQueue.h` 和 `ApplicationQueues.h`，提供 History、Alarm、OperationLog、MatrixFrame 的线程安全队列。
- 新增 `application/workers/BatchPersistWorker.h`，支持 start/stop、间隔 flush、批量 flush、显式 flush 和 shutdown flush。
- 新增 `application/runtime/DataSourceHealthMonitor.*`，实现 WaitingForFirstFrame、Online、TimedOut、Stopped 状态和超时事件。
- 新增 `application/runtime/MonitoringRuntimeService.*`，实现 RawFrame -> 清洗 -> 报警评估 -> 事件发布 -> 超时 Offline 状态链路。
- 新增 `application/runtime/RuntimeLifecycleCoordinator.*`，封装运行时线程 start/stop/join/faulted。
- 新增 `application/runtime/PersistenceRuntimeCoordinator.*` 和 `AcquisitionRuntimeController.*`，为阶段 7 SQLite worker 接入预留统一控制面。
- 新增 `application/services/RuntimeEventConsumers.*` 和 `OperationLogService.*`，迁移 TagCache、History、Alarm、OperationLog、MeasurementMap 事件消费者。
- 更新 `simulator/adapters/SimulatorDataSource.*`，实现 `IRawFrameSource`，供运行时以抽象方式读取帧。

## 行为对齐

- 事件发布按订阅 order 顺序执行。
- Isolated handler 失败会记录错误并继续后续 handler；Critical handler 失败会中断发布并返回失败。
- RawFrameReceivedEvent 会更新 MeasurementMapService。
- TagRuntimeStatesProducedEvent 会更新 TagCache，并按 TagRuntimeConfiguration 写入 HistorySampleQueue。
- AlarmRaised / Updated / Recovered / Acknowledged 会进入 AlarmEventQueue，并写 OperationLogQueue。
- DataSourceTimedOut / Recovered 会写 OperationLogQueue。
- 数据源超时后保留上次值，但将质量置为 Offline，并重新发布运行时状态。
- BatchPersistWorker 在批量、间隔、显式 flush 和停止时都会 flush 内部缓冲。
- RuntimeLifecycleCoordinator 可启动运行时线程，并在 stop 时请求取消和 join。
